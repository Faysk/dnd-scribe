# Guia da Demo Visual — DnD Scribe

Esta demo foi feita para discussão visual com a mesa. Ela não é o sistema final, mas sim uma maquete navegável para decidir o que vale construir primeiro.

## Arquivos principais

- `pitch.html`: página curta para apresentar a ideia.
- `index.html`: demo navegável com as principais telas.
- `assets/css/styles.css`: visual completo.
- `assets/js/app.js`: dados mockados e interações.

## Roteiro para apresentar para a mesa

### 1. Abrir `pitch.html`

Use como introdução visual. Explique a ideia central:

> A sessão inteira vira arquivo. Só o que sobrevive à revisão vira história.

### 2. Abrir `index.html`

Comece pela tela **Agora**. Ela mostra como seria o dashboard de uma sessão ativa, com recaps, relógios narrativos e itens importantes.

### 3. Ir para **Captura**

Mostre como as fontes seriam organizadas:

- Craig para áudio multitrack.
- OBS como backup.
- Roll20 Pro Logger para eventos estruturados.
- Discord para marcadores rápidos.

### 4. Ir para **Revisão**

Essa é a tela mais importante. Mostre que a IA não publica nada sozinha. Ela cria candidatos, e a mesa aprova, rejeita ou corrige.

### 5. Ir para **Canon**

Mostre o quadro de decisão narrativa:

- Candidato.
- Canon aprovado.
- Interpretação.
- Publicado.

### 6. Ir para **Bastidores**

Explique que conversa aleatória não precisa poluir a lore, mas pode virar corte divertido se todo mundo aprovar.

## Perguntas para a mesa

- Quais telas parecem essenciais?
- Quais telas parecem firula bonita, mas dispensável?
- Quem deve poder aprovar canon?
- Jogadores podem revisar somente suas próprias falas?
- Bastidores devem existir ou é melhor manter tudo privado?
- O que deve ser público no site final?
- O que deve ser exclusivo do mestre?

## MVP recomendado depois da demo

1. Criar sessão.
2. Upload de arquivos Craig e Roll20.
3. Transcrição por faixa.
4. Tela de revisão.
5. Aprovação de canon.
6. Recap publicado.

Não comece por grafo, música, dashboard perfeito ou features bonitas demais. Primeiro resolve o arroz com feijão: registrar sem virar bagunça. Depois mete o tempero do bardo.
