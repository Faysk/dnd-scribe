# Regras de visibilidade e segredos

## Conceitos separados

### Quem vê no sistema
Permissão real do app.

Exemplos:

- só DM;
- só Renan;
- Renan + DM;
- Renan + Arthur;
- grupo todo;
- público.

### Quem sabe dentro da ficção
Conhecimento narrativo.

Exemplos:

- Dandelion sabe;
- Astel sabe;
- Screaky não sabe;
- o povo de Euclix acredita em uma mentira;
- o jogador sabe fora de personagem, mas o personagem não.

## Tipos principais

### Diário privado
Visível só para o jogador. Não é canon. Não alimenta IA. Não aparece para o DM.

Uso: pensamentos, teoria pessoal, frase futura, sentimento interno, rascunho.

### Segredo de personagem
Visível para jogador + DM. Pode afetar canon e gerar consequências.

Uso: descoberta secreta, decisão oculta, pacto, item escondido, backstory relevante.

### Segredo compartilhado
Visível para players específicos + DM.

Uso: plano entre dois personagens, informação revelada só para parte do grupo.

### Segredo do DM
Visível só para o DM.

Uso: preparação, vilões, pistas ocultas, revelações futuras.

### Canon público
Visível para a mesa toda. Pode entrar em recap e timeline.

### Bastidor
Não canon. Pode ser privado ou publicado como corte aprovado.

## Regra de ouro

Segredo sem DM é diário. Segredo com DM é munição narrativa.
