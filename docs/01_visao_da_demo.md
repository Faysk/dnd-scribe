# Visão da Demo v2

Esta demo serve para mostrar para a mesa como o DnD Scribe pode funcionar antes de construir o backend real.

O foco não é só transcrever sessões. O foco é transformar uma sessão longa de D&D em uma memória auditável, revisável e controlada por perspectiva.

## Problema resolvido

Campanhas longas têm vários tipos de informação:

- fatos públicos;
- decisões privadas;
- segredos entre personagens;
- segredos do DM;
- teorias de jogador;
- diários internos;
- conversas aleatórias;
- falas marcantes;
- bastidores engraçados;
- eventos mecânicos do Roll20.

Sem um sistema de visibilidade, tudo vira uma sopa. E sopa de lore é boa até cair metagaming dentro.

## Solução proposta

Cada informação deve ter:

- fonte;
- timestamp;
- dono;
- status de canon;
- visibilidade no sistema;
- conhecimento dentro da ficção;
- histórico de revelação.
