A ponte usa um script no Roll20 para gerar pacotes tecnicos e uma extensao do navegador do GM para enviar esses pacotes para o DnD Scribe. Jogadores nao precisam ver nem digitar token.

Camada 1 Mod/API Roll20

Roda no Roll20 e transforma chat, rolagens e comandos em pacotes tecnicos.

Camada 2 Extensao do GM

Le os pacotes no navegador, esconde o texto tecnico e envia para producao.

Camada 3 Timeline

Eventos entram em roll20_events para sincronizar com audio e Discord.

Seguranca Token dedicado

A API aceita somente ROLL20_BRIDGE_TOKEN. Sem ele, nada e gravado.

1. Pausar antes de configurar

No chat do Roll20, rode este comando antes de mexer na extensao. Isso evita spam tecnico para o GM.

!dndscribe off
2. Instalar ou atualizar Mod/API

Abra a pagina do jogo no Roll20, va em Settings -> Mod (API) Scripts -> New Script, cole o script e salve. Use sempre a versao publicada abaixo.

Abrir script Mod
3. Instalar extensao Chrome local

No Chrome, abra chrome://extensions, ative Developer mode, clique Load unpacked e selecione a pasta local da extensao.

D:\Projects\dnd\integrations\roll20\chrome-extension
Ver manifest
4. Copiar token e sessao alvo

Carregando configuracao segura...

5. Configurar no Roll20 editor

Abra a mesa em https://app.roll20.net/editor/.... No painel pequeno DnD Scribe, clique em Config e cole URL, campaign slug, sourceSessionId e token.

URL: https://dnd.faysk.dev
Campaign: yuhara-main
6. Testar e ligar

Depois de configurar, confira status, ligue a ponte e envie uma acao teste.

!dndscribe status
!dndscribe on
!dnd acao teste da ponte roll20
Emergencia Se aparecer texto gigante DND_SCRIBE_EVENT para o GM

Pause imediatamente, recarregue a extensao e a mesa, confirme que o painel DnD Scribe apareceu e configure novamente antes de religar.

!dndscribe off
