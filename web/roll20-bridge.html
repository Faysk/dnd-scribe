Fluxo seguro Deixe o legado quieto, configure a extensao e monitore a fila.

A ponte usa a extensao do navegador do GM como captura principal. O Mod/API fica como apoio legado e deve permanecer com transporte por chat desligado. Jogadores nao precisam ver nem digitar token.

Camada 1 Extensao do GM

Observa chat, rolagens e comandos no navegador e envia lotes para producao.

Camada 2 API DnD Scribe

Valida o token dedicado, normaliza eventos e grava em roll20_events.

Camada 3 Timeline

Eventos entram em roll20_events para sincronizar com audio e Discord.

Seguranca Token dedicado

A API aceita somente ROLL20_BRIDGE_TOKEN. Sem ele, nada e gravado.

1. Deixar legado quieto

No chat do Roll20, rode estes comandos para garantir que scripts antigos nao enviem pacotes tecnicos pelo chat.

!dndscribe transport off
!dndscribe off
2. Instalar ou atualizar Mod/API

Abra a pagina do jogo no Roll20, va em Settings -> Mod (API) Scripts -> New Script, cole o script e salve. Use sempre a versao publicada abaixo.

Abrir script Mod
3. Instalar extensao Chrome local

No Chrome, abra chrome://extensions, ative Developer mode, clique Load unpacked e selecione a pasta local da extensao.

D:\Projects\dnd\integrations\roll20\chrome-extension
Ver manifest
4. Copiar token e sessao alvo

Carregando configuracao segura...

5. Configurar no Roll20 editor

Abra a mesa em https://app.roll20.net/editor/.... No painel pequeno DnD Scribe, clique em Config e cole URL, campaign slug, sourceSessionId e token. O painel deve mostrar Captura: DOM direto.

URL: https://dnd.faysk.dev
Campaign: yuhara-main
6. Testar e monitorar

Depois de configurar, envie uma acao teste. Use status apenas para diagnosticar o Mod legado; nao use transport on no fluxo normal.

!dndscribe status
!dnd acao teste da ponte roll20
Emergencia Se aparecer texto gigante DND_SCRIBE_EVENT para o GM

Desligue o transporte legado, remova scripts antigos duplicados, atualize o Mod e recarregue a extensao/mesa. O fluxo normal deve ficar em DOM direto.

!dndscribe transport off
